Zuma Rock — Nigeria (site plan)

Site name

Zuma Rock — Nigeria — a focused, single-page student site that highlights Zuma Rock (near Abuja). I chose this name because it directly describes the page subject and makes the purpose clear to visitors.

Files & folders (naming conventions): Use lowercase, no spaces: project/, siteplan.html, place.html, styles/place.css, scripts/place.js, images/.

Site purpose

This site will provide basic information about Zuma Rock: location, short facts, a responsive hero image, and a small weather/data card. It demonstrates responsive HTML/CSS and simple JavaScript (dynamic footer dates and a wind-chill calculation as required by the course).

Scenarios

  • Visitor: “How do I get to Zuma Rock from Abuja and what can I expect?”
  • Student: “What are the basic facts (coordinates, state, and why it's famous)?”

Primary content

  • Responsive hero image (3 sizes, WebP)
  • Data card (capital, coordinates, short facts)
  • Weather card with static inputs, windchill calculation via JS
  • Footer with copyright year and last modified (via JS)

HTML structure & JavaScript

Student-level structure to implement (example):

<header>site title</header> <main> <section class="hero"><picture>...</picture></section> <section class="info-grid"> <article class="data"></article> <aside class="weather"></aside> </section> </main> <footer>copyright + lastModified (via JS)</footer>

JavaScript notes:

  • Use an external `scripts/place.js` file.
  • Set `document.getElementById('currentYear').textContent = new Date().getFullYear()` for copyright year.
  • Use `document.lastModified` and `new Date(document.lastModified).toLocaleString()` to show the last modified date (note: when testing locally file timestamps can be empty; use Live Server or GitHub Pages for accurate values).
  • Include a simple `calculateWindChill(tempC, windKmh)` function and only call it if `tempC <= 10 && windKmh > 4.8` (per rubric).

Testing & self-checklist

  • Run W3C HTML validator for place.html (no errors).
  • Run CSS validator for styles/place.css (no errors).
  • Open site with Live Server and check DevTools Console (no JS errors).
  • Run DevTools Lighthouse (Desktop & Mobile) for Accessibility / Best Practices / SEO.
  • Hard reload (Empty Cache) and verify transferred bytes ≤ 500 KB (optimize images to meet this).
  • Check color contrast with DevTools CSS Overview or another color-contrast tool.
